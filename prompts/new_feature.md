# New Feature Prompt

Implement a new feature in this Flutter repository.

Before coding:
1. inspect the existing repo structure
2. follow repo conventions
3. follow `AGENTS.md`, `CLAUDE.md`, `agents/shared/*`, `design/DESIGN.md`, and relevant handoff files

For the feature, identify:
- user goal
- route entry
- remote data needed
- local data needed
- states required
- failure modes
- cache behavior

Requirements:
- use presentation/domain/data
- no Dio in widgets
- no JSON parsing in widgets
- localize user-facing text
- add verification before finishing
